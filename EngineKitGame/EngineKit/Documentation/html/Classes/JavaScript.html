JavaScript Class Reference

Inherits from NSObject
Declared in JavaScript.h
JavaScript.m

Overview

A JavaScript manager class, used to handle the interaction between the Objective-C game architecture and the JavaScript code.

+ shared

The singleton instance of the JavaScript class. The class is meant to be used as a singleton and deals with other singleton instances, such as the SCNScene and the JSContext singletons.

+ (JavaScript *)shared

Return Value

A singleton instance of the JavaScript class.

JavaScript.h

– initWithFile:

Initializes all needed JavaScript data and runs the file determined by the given @p filename, executing the code in its global scope and setting up default functions (such as @p load and @p update).

- (instancetype)initWithFile:(NSString *)filename

Parameters

filename

A path to the file, relative to the project’s folder. If the file is in the project’s folder itself, this string would only be the file’s name.

Return Value

An initialized JavaScript object set up to handle the given file.

JavaScript.h

– setup

Sets up the framework for use in the Javascript code, runs the global code and loads the necessary functions (such as @p load and @p update) so that they are ready to be called.

- (void)setup

Discussion

Warning: this method is called automatically by @p -initWithFile and @p init, so there should be no need to call it manually.

– contactCallback

Returns the current callback function for contact handling in javascript. It should be a function called “contact” or a variable, with the same name, that contains the function to be called.

- (JSValue *)contactCallback

Return Value

An JSValue, hopefully containing the callback function to be called (if the JavaScript code is correct); if there is no such function or variable, the JSValue returned will be undefined.
